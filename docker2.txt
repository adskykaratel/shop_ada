Docker  - это платформа для разработки , позволяет запускать приложения в контейнерах
Контейнеры представляют собой изолированные среды , в которых запускаются приложения и их зависимости

images (образы )- шабланы из которых создаются контейнеры  
Container(контейнеры) - это экзампял образа , который запушен в определнный момент времени 
Volues(том) - это механизм хранения данных в Docker
Docker Engine - это движок , который управляет контейнерами 
Docker Compose - инструмент для определения и запуска многоконтейнерных приложении 

docker ps - показывает текущие конттейнеры которрые работают 
docker images - показывает наши установленные образы
 у контейнара есть свой айди своя команнда когда был создан и статус и порт а так же имя 
 для нас будет лучш е работать с айдишками
  с командай stop мы может останавить это надо указать по айди 

  docker exec - it 04c773e6129a bash
  там вы указываем название нащего контейнера айдишку 
  удаление контейнера docker rm и айдишка , нельзя удалить когда контейнер работает
  есть команда remove
  and kill он сразу убивает даже тогда контайнер запушен

docker ps -a - показывает все Контейнеры


docker start 'CONTAINER_ID' - останавливает 
docker kill 'CONTAINER_ID' - убивает
docker rm 'CONTAINER_ID' - удаляет
docker rmi 'name_images' - удаляет образ 
dcker exec -it 'CONTAINER_ID' bash - запускает терминал запущенного контейнера 


Dockerfile:
from - главный образ с которог идет сборка ващего контейнера 
MAINTAINER name <mail>- кто     собрал образ
RUN - запускает команды
CMD - запускает проццесы

COPY - копирует файлы из нашего хоста во внутрь контайнера(. /urs/src/app) - первым указываем хост , вторым контайнер

EXPOSE - открывает порт 
WORKDIR - устанавливает рабочую директорию . мы попадаем в нее при заходе внутрь контайнера
